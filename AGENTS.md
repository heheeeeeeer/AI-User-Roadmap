# AI Roadmap 开发指南

本文件是本项目的 Codex 强提示文件。每次进入本仓库处理代码、样式、内容或构建配置前，都必须先阅读本文件，并按其中约束执行。

## 每次改动前

1. 先阅读 `DEVELOPMENT_LOG.md`，确认上一次改动内容、遗留风险和回归检查点。
2. 如果本次改动会影响上一次改动覆盖的功能，必须主动检查旧问题是否复现。
3. 不要只按新需求修改；需要同时保护已经确认过的体验，包括 Astro 构建、竖向路线图布局、资源默认收起、无横向溢出、手写注释不遮挡内容。

## 每次改动后

1. 必须在 `DEVELOPMENT_LOG.md` 新增一条记录。
2. 记录至少包含：日期、改动摘要、影响文件、已做验证、下一次改动前需要防止复现的问题。
3. 如果无法验证某项内容，必须在记录里明确写出原因。

## 项目约定

1. 当前项目是 Astro 静态站点，源码以 `src/` 为主。
2. 路线图数据放在 `src/data/roadmap.js`。
3. 页面结构组件放在 `src/components/`。
4. 全局视觉样式放在 `src/styles/global.css`。
5. 公开静态资源放在 `public/`。
6. 默认使用 `npm run build` 做最终验证。
7. 需要本地预览时使用 `npm run dev -- --port 8765`。
